# ANG HR

ANG HR 是手機優先的人力資源管理介面，目前提供員工與企業管理端的 Beta 操作雛形，包含日夜模式、排班、出勤、請假、審核、人員及薪資等流程。

## 目前版本定位

目前工作原始碼已凍結為 **`BETA-v0.3`**，等待手動推送。預定朋友測試網址為 <https://edn869728-jpg.github.io/ANG-99-HR-System/beta/v0.3/>，但尚未確認上線；收到發布確認前請勿將此網址視為可用測試版。此版適合進行介面、操作流程、方案差異與手機版測試，但仍不是正式營運系統。已交付的 `BETA-v0.1` 與 `BETA-v0.2` 凍結目錄、ZIP、SHA-256 及既有回報均保留不變，不會被 v0.3 覆蓋。

- 所有測試資料只保存在目前瀏覽器的 `localStorage`。
- 不同裝置及不同瀏覽器之間不會同步資料。
- Email 驗證碼、Google、LINE、企業帳號及個人帳號尚未連接真實身分驗證服務；BETA-v0.3 入口只接受文件列出的 7 組固定臨時測試資料。
- 目前沒有正式 API、資料庫、檔案儲存、Token 驗證或伺服器端權限檢查。
- 檔案上傳目前只模擬操作流程，不會將實際檔案傳到伺服器。
- 同一分頁重新整理後會保留目前臨時測試身分；選擇切換帳號或登出後才會回到入口。瀏覽器內已建立的 Demo 資料仍會保留。
- 此版本沒有正式帳號鎖定、停權、方案到期或登入嘗試限制；臨時帳號只用於角色、方案導覽與流程測試。
- 請勿輸入真實個資、薪資、公司機密或正式文件。

## BETA-v0.3 方案測試矩陣

- 企業：Business Basic、Business Pro、Business Premium，各有獨立臨時帳號與可用頁面／額度提示。
- 個人：Personal Solo、Personal Performance，各有獨立臨時帳號；兩者都不提供設定入口。
- 免費：Personal Lite 進入精簡個人工作區；Business Lite 進入精簡小團隊管理工作區。
- 7 組帳號會顯示各自方案名稱、額度、包含功能與未包含功能；完整憑證見 `BETA_TEST_GUIDE.md`。
- Business Premium 方案包含進階報表、自訂角色與安全稽核，但這三項在目前 Beta 尚未串接。
- Personal Performance 方案包含目標 KPI、績效回饋與績效分析，但這三項在目前 Beta 尚未串接。
- 方案差異目前仍由前端 Demo 控制，不能視為正式伺服器端訂閱或權限驗證。

## 啟動

建議使用 lockfile 安裝目前已驗證的相依版本：

```bash
npm ci
npm run dev
```

正式建置：

```bash
npm run build
```

建置完成後，可部署的靜態檔案位於 `dist/`。

## 目前可測試流程

- 入口頁：日夜背景、免費／個人／企業入口與方案展示
- 員工端：首頁、班表、打卡、請假、薪資明細、資料上傳與留言互動
- 管理端：營運總覽、審核、排班、企業設定、公告、人員與薪資管理
- 本機持久化：打卡、請假、留言、審核、人員及部分設定保存在 `localStorage`
- 響應式版面：手機滿版，桌面以手機框呈現

## 未來正式架構

正式版本將依下列邊界建置，不再採用 Base44 方向：

### Logto：唯一身分驗證來源

- Email OTP、Google、LINE 與其他登入方式統一由 Logto 管理。
- 前端不自行判定登入成功、角色或公司權限。
- 受保護 API 必須驗證 Logto 簽發的有效 Token。
- 停權、Token 過期、登出及裝置工作階段必須由正式驗證流程處理。

### Supabase：資料、檔案與即時更新

- PostgreSQL 儲存公司、人員、分店、排班、請假、打卡、薪資及公告等業務資料。
- Supabase Storage 儲存經授權的 Logo、頭像、證明文件及其他附件。
- Supabase Realtime 僅用於需要即時同步的資料更新。
- Supabase 不取代 Logto，也不作為第二套使用者登入來源。

### 嚴格公司隔離

- 每一筆公司範圍資料都必須包含不可省略的 `company_id`。
- 所有查詢、修改、刪除、檔案路徑及 Realtime 訂閱都必須限制在目前授權公司。
- 公司隔離必須同時由資料庫 RLS 與受信任的伺服器 API 驗證，不能只依靠前端篩選。
- 角色與分店權限必須由伺服器依 Token、公司成員資格及權限規則判斷。
- 員工不得進入管理端；管理者也不得讀取未授權公司的資料。
